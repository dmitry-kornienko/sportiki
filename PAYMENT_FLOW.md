# Флоу подтверждения оплаты

## Статус: в разработке (фронт-заглушка готова)

Кнопка "Оплатить" и форма загрузки скрина уже есть в EventDetail.
Логика отправки — заглушка, бэкенд не реализован.

---

## Поле в таблице Registrations

Добавить столбец `PaymentStatus` (не boolean):

| Значение | Смысл |
|---|---|
| `''` | оплата не требуется / скрин не отправлен |
| `PENDING` | скрин загружен, ждёт подтверждения админа |
| `CONFIRMED` | оплата подтверждена |

---

## Флоу

```
Пользователь загружает скрин в Mini App
  → POST ?action=submit_payment { eventId, fileId }
    → Sheets: PaymentStatus = PENDING
    → Бот → Админу:
        Фото (скрин перевода)
        "👤 @username оплатил Велозаезд — 590 000 ₫"
        [Кнопка: Подтвердить оплату]
          → Callback в боте
            → Sheets: PaymentStatus = CONFIRMED
            → Бот → Пользователю: "✅ Оплата подтверждена"
```

---

## Вопрос перед реализацией

Скрин отправляется прямо из Mini App или через бот?
- **Из Mini App** — нужен эндпоинт принимающий файл, затем бот пересылает его админу
- **Через бот** — пользователь переходит в бот и отправляет фото там напрямую

Это влияет на то как файл попадает в Telegram (file_id).
