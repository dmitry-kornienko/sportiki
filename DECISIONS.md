# Архитектурные решения

Здесь фиксируем решения по стеку и архитектуре, чтобы не возвращаться к одним вопросам дважды.

---

## 2026-05-08 — Остаёмся на GAS + Sheets

**Решение:** не мигрировать на Vercel + Supabase + Node.js до полной готовности продукта.

**Контекст:** была рассмотрена альтернативная архитектура (React/Next.js → Vercel, NestJS serverless, Supabase PostgreSQL, GAS только для крон).

**Почему отказались сейчас:**
- Текущий стек полностью рабочий
- Переписывание инфраструктуры до готовности продукта = время на инфраструктуру вместо фич
- Этапы 2-4 из CLAUDE.md ещё не завершены (события в Mini App, сканер, QR-билеты)

**Триггеры для пересмотра:**
- Sheets начинает лагать (обычно при 3000–5000 строк в Registrations)
- Нужны сложные запросы (аналитика, joins, фильтры)
- Команда разработки растёт

## Целевая архитектура (когда придёт время)

```
Telegram Bot (GAS)        ← оставить навсегда: бесплатный крон + уведомления
       ↓
Vercel
  ├── React Mini App       ← замена GitHub Pages
  └── /api/* (Serverless)  ← замена GAS API, Node.js/TypeScript
       ↓
Supabase (PostgreSQL)      ← замена Google Sheets
```

**Заметки по Supabase Free Tier:**
- Проект засыпает после 7 дней неактивности (~30-60 сек на пробуждение)
- Для Sportiki не проблема: ивенты каждый день, неактивности не будет
- Лимит: 500 MB хранилища, паузится на бесплатном плане
- Платный план (убрать паузу + больше места): $25/мес

**Порядок миграции (по частям, не всё сразу):**
1. GitHub Pages → Vercel — быстро, ~1 час, без переписывания кода
2. Google Sheets → Supabase — мигрировать данные, обновить Database.js
3. GAS API → Vercel Serverless Functions — переписать контроллеры на Node.js/TS
4. GAS Bot — трогать последним или оставить как есть навсегда
